# Screen Specifications

No UI screens are authorized or generated in the harness milestone.

## Future specification template

For each phase-approved screen, document:

- user outcome and entry/exit criteria;
- required role/permission and organization context;
- real data contract and provenance;
- loading, empty, partial, stale, error, offline/retry, and forbidden states;
- mobile/tablet/desktop behavior;
- keyboard order, semantics, announcements, focus, and contrast;
- analytics/audit events without sensitive payloads;
- destructive or high-impact confirmations;
- linked acceptance and security tests.

Illustrative content must be visibly labeled and cannot substitute for a working tenant-scoped data path.
