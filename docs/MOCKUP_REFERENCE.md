# Mockup Reference

No mockup or screenshot is treated as implemented product behavior in this harness.

## Future handoff contract

When approved mockups exist, record:

- canonical Figma file and node links;
- design version and review date;
- corresponding user story and active phase;
- responsive variants and accessibility annotations;
- data fields, provenance, permission rules, and all non-happy states;
- component and token mappings;
- explicitly non-functional illustrative content.

## Implementation rule

Mockups communicate intent but do not override business rules, authorization, RLS, security requirements, or acceptance tests. Screenshots belong in `figma-handoff/screenshots/` only when licensed, reviewed, and useful for an active implementation. The directory is intentionally empty.
